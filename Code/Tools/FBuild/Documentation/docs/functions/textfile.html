TextFile

TextFile

Creates a text file as a build step. The text file can then be used as part of a subsequent build step, though this TextFile node may need to be listed explicitly. For example. when used as part of an ObjectList .CompilerOptions or scanned via wildcard, it should be in the ObjectList's .PreBuildDependencies.

The destination directory will be created automatically, if needed.

By default, the text file will be updated if it is missing or the BFF has changed.

.TextFileInputStrings must contain at least one string, and none of the strings can be empty. To output a blank line, you must write a space.

TextFile( alias ) ; (optional) Alias { .TextFileOutput ; Output file to generate .TextFileInputStrings ; Array of non-empty strings to be written to the output file, one per line. .TextFileAlways ; (optional) Generate the file even if it already exists with the current contents. ; Additional options .Hidden ; (optional) Hide a target from -showtargets (default false) .PreBuildDependencies ; (optional) Force targets to be built before this TextFile (Rarely needed, ; but useful if the output would be deleted by an earlier step.) }
